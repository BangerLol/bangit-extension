# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.3.x   | Yes       |
| < 1.3   | No        |

## Reporting a Vulnerability

If you discover a security vulnerability in the Bangit extension, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

1. **Email**: Send details to billy.banger.mail@gmail.com
2. **Subject line**: `[SECURITY] Brief description of the vulnerability`
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: Within 7 days, we'll confirm the vulnerability and its severity
- **Fix timeline**: Critical vulnerabilities within 72 hours; others within 30 days
- **Disclosure**: We'll coordinate disclosure timing with you

### Scope

The following are in scope:
- The Bangit Chrome extension (this repository)
- Authentication and token handling
- Data sent to/from the Bangit backend API
- Content script injection behavior on twitter.com/x.com

The following are out of scope:
- The Bangit backend API (report separately)
- Twitter/X platform vulnerabilities
- Social engineering attacks

### Recognition

We appreciate security researchers who help keep Bangit safe. With your permission, we'll acknowledge your contribution in our release notes.
